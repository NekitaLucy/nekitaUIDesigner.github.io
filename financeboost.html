Finance Boost

Timeframe

April 2023 - May 2023

Role:

Research, ideation, and design

Skills applied for this project:

Building a cohesive brand, User Flow Diagram, Wireframes, User testing notes and results, UI Design, Desktop Design, Mockup, and Prototyping.

Tools: Figma and Balsamiq

Save money, track finances, and follow up on your financial goals.

Purpose

Design a money-saving app for a responsive website and mobile app based on the following brand description.

Brand Description

Users will input personal and financial information, so they want to use a trustworthy product. The product should be considered reliable, serious and build confidence in its users. Appealing to those less familiar with the world of finance, so we must convey simplicity and clarity through our brand.

Process

The key principles behind the brand are the brand description and key messaging.

The designs and components of the brand.

  • Guiding Principles
  • 5 Ws: who, what, where, when and where
  • Logo
  • Colors
  • Typography
  • Image Style (photographs, illustrations, patterns, etc.)
  • Writing style

Guiding Principles

Our brand guidelines are centred around a clean, dark, and light blue palette that conveys reliability, trust, and professionalism. The core of our brand is simplicity and clarity. We ensure brand identity consistency across all platforms and channels. Our brand makes money-saving easy, approachable, and understandable through a personalized financial plan that motivates you to achieve your goals. We provide accurate, reliable tools and resources and communicate our expertise and knowledge of personal finance.

User flow design

Using the three user stories from theproject brief, I employed a user flow design process to create a seamless and intuitive user experience.

Through user research and interviews, I began by understanding the target audience's financial goals, pain points, and behaviours.

Using this knowledge, I mapped out the user flow, identifying key actions such as setting savings goals, tracking expenses, and receiving personalized recommendations.

Wireframing

Selecting 1 key flow from the user flow diagram, I created the low- and mid-fidelity wireframes using first pen and paper.

User feedback and iterations to refine the design

I recruited 3 to 5 participants, recorded, and implemented their feedback. I was testing the designs for a user flow using a scenario-based prototype test. I applied the feedback to the designs and updated the high-fidelity grayscale wireframes.
